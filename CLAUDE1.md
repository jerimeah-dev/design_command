# design_command — Project Rules

These rules apply to all work done in this project and to all Flutter code generated by the commands in `.claude/commands/`.

---

## State Management

1. **No `context.watch`** — use `Consumer<T>` or `Selector<T,R>` for reactive UI; `context.read<T>()` for event handlers only (triggers no rebuild).
2. **`provider` is the only state management package** — do not use Riverpod, Bloc, GetX, or any other package.

## File Structure

3. **1 file per class** — `StatefulWidget` and its `State` class share the same file (they count as one logical unit).
4. **File naming — feature files**: `{className_snake}.{folderName}.dart`
   - Example: `CreatePostPage` → `create_post_page.create_post.dart` (inside `create_post/` folder)
   - Example: `MediaPickerWidget` inside `create_post/` → `media_picker_widget.create_post.dart`
5. **File naming — shared widgets**: `{className_snake}.dart` inside `lib/widgets/`
   - Example: `ThemeDropdown` → `lib/widgets/theme_dropdown.dart`
6. **Folders**: always `snake_case`
7. **Classes**: always `PascalCase`

## Theme System

8. **Theme is global** — one `ThemeProvider` at root (`app.dart`); all pages react instantly via `Consumer<ThemeProvider>`.
9. **No hardcoded colors** — always use `Theme.of(context).colorScheme.*`. Never use `Color(0xFF...)` inside widget trees.
10. **`ColorScheme.fromSeed()`** for all themes — never manually assemble palettes from hex values.
11. **Themes are modern, minimal, low-saturation** — use neutral/desaturated seed colors only.
12. **Themes must be consistent with `design_principles.md`** (Material Design 3).

## Versioning

13. **Each command run creates a new version folder** (`lib/versions/vN/`) — existing versions are **never** modified or deleted.
14. **`VersionSwitcher` widget auto-hides** when `totalVersions == 1`; shows chip row (V1, V2…) when `totalVersions ≥ 2`.

## Layout & Typography

15. **No hardcoded font sizes** — always use `Theme.of(context).textTheme.*`.
16. **8dp spacing grid** — all padding and margins are multiples of 8dp (use 4dp only for fine adjustments within components).
17. **Adaptive navigation**: `NavigationBar` (< 600dp), `NavigationRail` (600–1200dp), `NavigationDrawer` (> 1200dp) — switch at runtime using `LayoutBuilder`.

## Accessibility

18. **Minimum 48×48dp touch targets** — wrap smaller visual elements in `SizedBox` if needed.
19. **Semantic labels** on all interactive and meaningful elements (`Semantics`, `IconButton(tooltip:)`, etc.).
20. **Respect `MediaQuery.disableAnimations`** — skip or snap animations when it is `true`.
